# outputs/pr-descriptions/

This directory stores GitHub pull request descriptions generated by the Supervisor Agent.

Each file is named `<work-item-id>_PR.md` and corresponds to one pipeline run.

The PR description is also posted directly as the PR body on GitHub when the Supervisor
creates or updates the pull request.

**Template:** [`pipeline/templates/pr-description-template.md`](../../pipeline/templates/pr-description-template.md)

> This directory is gitignored in production. Contents are pipeline-generated.
